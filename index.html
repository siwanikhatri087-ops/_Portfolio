Section 1

What is Full Stack Development?

Full stack development refers to the practice of building both the front-end (client side) and back-end (server side) of a web application. A full stack developer works across the entire technology stack — from databases and servers, to APIs, to the user interface that appears in a browser.

The Two Halves of the Stack

The front-end is everything a user sees and interacts with directly. It is built using HTML (structure), CSS (styling), and JavaScript (interactivity). Frameworks like Bootstrap, React, or Vue.js speed up front-end development by providing ready-made components and patterns.

The back-end runs on a server, invisible to users. It handles business logic, authentication, and data processing. Common back-end languages include Node.js, Python, PHP, and Java. The back-end communicates with databases (such as MySQL, PostgreSQL, or MongoDB) to store and retrieve information.

Between these two halves sits the API layer — typically RESTful APIs or GraphQL — which defines how the front-end and back-end talk to each other, usually by exchanging JSON data over HTTP.

Why Full Stack?

Full stack developers are highly versatile — they can prototype entire applications solo, communicate effectively between specialist teams, and understand how every layer of a system fits together, making them extremely valuable to employers.

⚙️

The Development Process

A typical workflow involves designing the UI wireframe, writing semantic HTML, styling with CSS, adding interactivity via JavaScript, connecting to a back-end API, and deploying to a cloud server (AWS, Heroku, Vercel, etc.).

📦

Modern Tooling

Full stack developers use version control (Git / GitHub), package managers (npm / pip), build tools (Webpack, Vite), and containerisation (Docker) to manage, automate, and deploy their codebases efficiently.

HTML is the backbone of every webpage. It provides the structural skeleton of a document using elements and tags, telling the browser what each piece of content is — a heading, a paragraph, an image, a form — independently of how it looks.

Section 3

CSS — Cascading Style Sheets

CSS controls the visual presentation of HTML elements — colours, fonts, spacing, layouts, and animations. Modern CSS is remarkably powerful, enabling complex responsive layouts and fluid animations without any JavaScript.

CSS Grid & Flexbox

CSS Grid is a two-dimensional layout system ideal for page-level structure. Flexbox handles one-dimensional layouts — rows or columns — perfectly. Together they replace the old float-based hacks and make responsive design straightforward.

Custom Properties (Variables)

CSS Custom Properties (--variable-name) allow you to define reusable values (colours, spacing) in one place and reference them everywhere, making themes and design systems easy to maintain.

Animations & Transitions

Using @keyframes and the animation / transition properties, CSS can produce smooth hover effects, fade-ins, slide-ins, and complex multi-step sequences — all without touching JavaScript.

CSS3 
:root {
  --accent: #06b6d4;
  --radius: 12px;
}

.card {
  display: grid;
  background: rgba(255,255,255,.1);
  backdrop-filter: blur(12px);
  border-radius: var(--radius);
  transition: transform .3s ease;
}

.card:hover {
  transform: translateY(-8px);
}

Bootstrap Framework

Bootstrap is the world's most popular open-source CSS framework. It provides a responsive 12-column grid system, pre-styled UI components, and utility classes that enable developers to build professional, mobile-first layouts rapidly.

The 12-Column Grid

Bootstrap's grid system divides the page into 12 columns. Using classes like col-md-4 (4 columns on medium screens) or col-lg-6, you can define exactly how many columns each element occupies at each breakpoint — automatically collapsing to full width on mobile.

Utility Classes

Bootstrap's utility system (mt-3, d-flex, text-center, bg-primary) lets you apply common styles directly in HTML without writing any custom CSS, drastically speeding up development.

Pre-built Components

Bootstrap ships with dozens of ready-made components — navbars, cards, modals, accordions, carousels, alerts, badges, and more — all styled consistently and accessible out of the box.

Section 5

JavaScript

JavaScript is the programming language of the web. It runs directly in the browser and enables dynamic, interactive experiences — responding to user actions, manipulating the DOM, fetching data asynchronously, and driving complex application logic.

DOM Manipulation

The Document Object Model (DOM) is JavaScript's interface to the HTML document. Using methods like document.getElementById(), querySelector(), and properties like textContent / innerHTML, JavaScript can read and modify any element on the page in real time.

Event Handling

JavaScript listens for user events — clicks, keyboard presses, form submissions, mouse movements — using addEventListener(). When an event fires, the associated callback function runs, making the page respond dynamically to user interaction.

State & Application Logic

Variables hold application state — the current values a program is tracking (e.g., the running total in a calculator). Functions encapsulate logic and can be called repeatedly. Together they form the foundation of interactive apps without needing a framework.

JavaScript ES6 
let count = 0;
const display = document
  .querySelector('#counter');

function increment() {
  count++;
  display.textContent = count;
}

document
  .querySelector('#btn')
  .addEventListener('click', increment);
